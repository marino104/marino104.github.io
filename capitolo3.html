Capitolo 1 - Introduzione

Benvenuti nel primo capitolo della nostra guida su GitHub. Qui esploreremo i concetti di base e l'importanza di GitHub per la gestione dei progetti di sviluppo software.

1. Cos'è GitHub?

GitHub è una piattaforma di sviluppo collaborativo che permette di gestire i progetti software utilizzando il sistema di versionamento Git. Permette agli sviluppatori di lavorare insieme, seguire il progresso dei progetti, e contribuire al codice sorgente in modo facile ed efficace.

2. Creare un nuovo repository

Per iniziare a utilizzare GitHub, è necessario creare un "repository". Un repository è una cartella in cui vengono archiviati i file del progetto. Qui potrai aggiungere, modificare o eliminare file, mantenendo traccia di tutte le modifiche effettuate nel tempo.

3. Clonare un repository

Per lavorare su un repository esistente, è possibile "clonarlo" sulla propria macchina locale. Questo ti permette di avere una copia del progetto per modificarlo offline.

Conclusione

Nel prossimo capitolo esploreremo come utilizzare i comandi Git per gestire il codice, fare commit, e gestire i branch. Prepara il tuo ambiente di sviluppo!
